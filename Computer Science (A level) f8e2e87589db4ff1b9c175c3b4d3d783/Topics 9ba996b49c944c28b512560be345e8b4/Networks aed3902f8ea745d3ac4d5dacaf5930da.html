Characteristics

It is two or more computer connected together with the ability to transmit data between each other

  • Types

    Local Area Networks (LANs)

    • A group of computers connected (network) which is spread over a small geographical area or a single site
    • A LAN uses connections owned by organisation
    • More secure than WAN
    • Advantages (over standalone computers)
      • Easier to maintain files and software
      • Sharing of peripheral devices
      • Monitoring of workers
      • Communication between machines

    Wide Area Networks (WANs)

    • A network which is spread over a large geographical area
    • A WAN often uses third party connection
    • Less secure than LAN
  • Connect WAN to LAN
    • A router, connect networks

Topologies

  • Physical topologies
    • The physical layout of the wires and components which form the network
    • Bus, star, mesh
  • Logical topologies
    • The layout show how data flows

Bus Topology (Bus network)

All devices are connected to a backbone cable, the ends of which are plugged into a terminator or a computer

  • Data is sent one at a time
  • One directional only
  • Data collision
    • More than one data is sent in the wire at the same time
  • Advantages
    • Relatively inexpensive to setup
    • Doesn’t require additional hardware
    • Device can easily be added
  • Disadvantages
    • Backbone is the point of failure (If it fails, the entire network fails)
    • As traffic increases, performance decreases (because of data collision)
    • All computer can see the data transmission (Data is sent to every computer in the network)

Star Topology (Star Network)

All devices are connected to a central node, often a switch.

  • Media Access Control (MAC) addresses are used to identify each devices on the network
  • The message is only sent to the computer it is specify for
  • Advantages
    • Performance is consistent even under heavy network traffic
      • No data collision
    • If a cable fails, only that single device connection is affected
    • Transmit data faster
    • (Easy to add new device)
  • Disadvantages
    • Expensive due to the switch
    • The central node is the point of failure (if the central node fail, the rest of the network fails)

Mesh Topology (Mesh Network)

All devices are connected to all other devices

  • Commonly use is wireless networks
  • Advantages
    • As the number of devices increase, the reliability and speed increase
    • Nodes are automatically incorporated
    • Node don’t go through central switch (speed increase)
  • Disadvantages
    • If using wired network, large quantity of cable is needed
    • Maintaining the network is difficult

Protocols

It is a set of rules defining how two computer communicate with each other / to govern data transmission.

  • Uses:
    • Allowing computers to communicate
    • Ensure all device follow the same rule
    • All device interpret the signal in the same way

  • HTTP (Hypertext Transfer Protocol)
    • Used for webpage rendering
    • HTTPS (Hypertext Transfer Protocol Secure)
  • TCP/IP (Transmission Control Protocol / Internet Protocol)
    • Used in routing of packages through the network
  • IP address (Internet Protocol)t
    • Set of four numbers identifying a resource on a network
  • POP3 (Post Office Protocol) + IMAP (Internet Message Access Protocol)
    • Mailing Protocols, used for email access
      • POP3 — downloads the message from a server to a single computer, the message is deleted from the server
      • IMAP — Stores the message on the server, synchronize the message across multiple devices
  • FTP (File Transfer Protocol)
    • Use for transmission of files over the network

The structure of the internet

Internet

It is a network of networks.

A public interconnection of computer networks allow data to be send globally to any connected device

Continents are connected to each other using large international backbone cables.

TCP/IP Stack and protocol layering

A stack of networking protocols that work together passing packets during communication

Application layer

  • Top of the stack
  • Specify what protocol is needed to be used in order to relate the application that is being sent
  • Specify what application the receiver should use
  • Example
    • HTTP
    • POP3
    • FTP

Transport layer

  • Uses TCP to establish an end-to-end connection between the source and recipient computer
  • It splits data up into packets
  • Labels with:
    • Their packet number
    • total number of packets
    • The port number being used for communication
  • If packets is lost, the transport layer requests retransmissions of these lost packets (check sum to check for errors)

Network layer

  • Add the source and destination IP address (The combination of the IP address and port number is socket address)
  • Routers operates on the network layer. It uses the IP address to forward the packets
  • The sockets are then used to specify which device the packets must be sent to and the application being used on that device

Link layer

  • It is the connection between the network devices, it adds the MAC address identifying the Network Interface Cards of the source and the destination computer
  • For devices in the same network, it will be the MAC address of the computer, otherwise, it will be the address of the router

It’s important to realise that this is a stack. On the recipient’s computer these layers are looked at from bottom to top.

Once the destination has been reached,

  1. the MAC address is removed by the link layer,
  1. then the IP addresses are removed by the Network Layer,
  1. then the transport layers remove the port number and reassemble + reorder the packets.
  1. Finally, the application layer presents the data to the recipient in the form it was requested in.

MAC address vs IP address

  • MAC address (Physical address)
    • A address unique to the device
    • Local identification
    • Operate within a LAN (Local Area Network)
    • Hardcoded (Cannot be changed)
  • IP address
    • A logical address
    • Global Identification
    • Assign through software configuration
  • Structure of MAC address and IP address

DNS (Domain Name System)

It is the system used to name and organise internet resources.

Smaller domain is separated from the larger domain by a full stop

  • Top Level Domain (TLD)
  • Second Level Domain (2LD)

Structure

When the IP address of a given domain is not known, it is referred to a related domain server that may know

  • Resolving IP address
    • Domain names used instead of IP address to reference a resource on a network
    • Local DNS serve contacted for IP address
    • If cannot supply one, send a request to another authoritative DNS (root servers)
  • How do local DNS servers know when new top-level domains are added
    • DNS names are hierarchical starting with the root servers
    • These root servers know the location of all top-level domains
    • As top level domains are added, the root server are updated
    • If local DNS server doesn’t have a record of an top-level domain, it contacts a root server, so it can be informed directly about any domain names in existence

URL (Uniform Resource Locator)

The combination of the protocol and fully qualified domain name of a resource

Used to specified the means of accessing a resource across a network and its location

Network Communication

Data packets

Packets are segment of data

  • Header
    • Sender and recipient IP address
      • Allow the packet to be delivered to the correct destination and trace the origin
    • Protocol being used
      • The protocol allow the recipient computer to know how to interpret the packets
    • Order of the packets + total number of packets
      • Allow the message to be reconstructed in the correct order
    • Time To Live / Hop Limit
      • Tells the packets when to expire so that it doesn’t not travel forever
  • Payload
    • Raw data to be transmitted
  • Trailer
    • Checksum / cyclic redundancy check
      • The trailer contains a code used to detect whether any errors have occurred during transmission.
    • End of packet flag

Circuit Switching and Packet Switching

Packet Switching

Data is sent using packets across the network

  • Packets are sent through the most efficient route individually
  1. Data is split into equal size packets
  1. Each packet has a header of information
    • Destination and source addresses and order of the packet in the whole message
  1. Each packet is dispatched to the internet through a router or gateway
  1. Packet are sent independently and may have different path
    • Router forward packets towards the destination code
  1. Packets need to be reordered and reassembled at the destination

  • Advantages
    • Multiple route can be used between devices, ensure the packet can be sent
    • Faster and more efficient data transfer
    • Safer from interception
      • As the packets uses different route, which makes it hard to intercept all the packets
    • Efficient use of the network as each channel is only used for a short time
    • If there is an error then only a packet is effected, only one packet is needed to retransmitted
  • Disadvantages
    • Time is spent deconstructing and reconstructing the packets
    • Must wait for all packets to arrive

Circuit Switching

A direct link is created between two devices

  • The link is maintained for the duration of the entire conversation
  • Advantages
    • Data arrive in logical order, which allow faster reconstruction
    • Enables two user to hold the conversation without delay
  • Disadvantages
    • Bandwidth is waste during period of time where no message is sent
      • The section of the network cannot be used by others until the full transmission is completed
    • Devices must transfer and receive data at the same rate
