Stages of software development

  • Analysis
  • Design
  • Implementation
  • Evaluation
  • Maintenance

Analysis

  • Interview people who will use the software
  • Use questionnaires to get information from large groups of people
  • Observe how the current system works
  • Look at exiting documentation

Produce system specification or user requirements

Design

  • A description of data: data type, format, and validation
  • Database design
  • Input screen
  • Output screen and reports
  • How the data will be processed
  • How the software will be tested

Implementation

  • Coding and testing the software
  • Writing user and technical documentation
  • Installing the software for the user
  • Testing

Evaluation

  • Acceptance testing
  • The product will be evaluated against the specification document

Maintenance

  • Corrective maintenance
    • Remove bugs
  • Adaptive maintenance
    • Adapt to changing user requirements
  • Perfective maintenance
    • Make the software faster, easier to use, more functionality

Testing

Done in implementation

Black box testing

Check the output against different input. The structure of the code doesn’t need to be known.

White box testing

It step through each path and the logic of the code

Alpha testing

Carried out by the developers and user

  • Reveal the errors or omissions in the system requirements
  • Test if it fits what the user wants

Beta testing

The software is given to potential users and reports any faults

Waterfall model

For:

  • Requirements are clear and fixed
  • Technology is well understood
  • Short project
  • Each stage is completed and documented before next is begun

Advantages

  • Simple to understand and use
  • Each stage is self contained with well defined outcomes and documentation

Disadvantages

  • Not much user involvement after Analysis stage
  • No working software until after implementation
  • Hard to make changes

Spiral Model

For:

  • Medium to high risk project
  • Requirements are unclear or complex
  • Large project that might need years to complete

Loop through: Analysis, Design, implementation, evaluation

  • Rapid prototyping

Advantages

  • Easy to manage
  • Software is produced in early stage → problems can be identified early
  • Any changes can be made early
  • Functionality can be added during the process
  • Easier to adapt to user requirements

Disadvantages

  • Time consuming (from getting feedback and refining)
  • Higher cost

Agile Model

For:

  • Small, time-critical project
  • Rapid incremental cycle
  • Each version builds on previous functionality
  • Each version is tested before release
  • Limited planning is needed

Advantages

  • Customer satisfaction due to rapid and continuous delivery of useful software
  • Constant interaction between customers, developers and testers
  • Easily adapted to changing circumstances
    • Late changes in requirements can be adapted

Disadvantages

  • Lack of emphasis on design and documentation
  • Fail to deliver if the customers is not clear about the desire final outcome
  • Not suitable for novice programmers

Rapid Application Development

Advantage

  • Higher user interaction and involvement
    • Changes can be made easily
  • Requirement doesn’t need to finished when the projects starts
