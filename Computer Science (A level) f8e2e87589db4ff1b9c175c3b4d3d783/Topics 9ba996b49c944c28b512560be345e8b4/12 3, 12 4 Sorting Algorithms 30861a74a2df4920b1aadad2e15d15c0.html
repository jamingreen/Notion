Questions

  • Describe how merge sort differs from a bubble sort
    • Merge sort splits the data
    • Merge sort sorts the split data as it is put back together
    • Bubble moves through the data in a linear way
    • Bubble moves through the data repeatedly
    • Merge is more efficient with large volumes of data to sort
    • Merge may require more memory space

  • Describe how quick sort algorithm works with the data (2, 0, 1, 7, 4, 3, 5, 6)
    • Uses divide-and-conquer
    • First item (2) becomes pivot
    • Compare each item to the pivot
    • Make two lists, 1 is less than the pivot and one is more then the pivot
    • Quick sort the new lists
    • recombine the sub-lists
    • Show the table

  • Compare the use of a merge sort and a bubble sort on this array, evaluating the performance of each sort, make reference the worst case big O notation
    • Knowledge
      • Merge sort uses sub-lists
      • Bubble sort uses a temp elements
      • Bubble sort moves through the list repeatedly
      • Merge sort divides the list into smaller lists
      • Merge is a recursive algorithm
      • Worst case of merge is logarithmic, scale up well
      • Worst case is exponential, does not scale up well
    • Application
      • It has small data set
      • Few changes are needed in this case
      • show use of merge and bubble
      • Calculation of average speed
    • Evaluation
      • Merge is fast on large data sets
      • Bubble is intuitive (easy to program)
      • Both are fast on smaller data
      • Bubble’s average speed worse than merge

  • Compare the use of merge, quick and insertion sort on an array with a small number of elements and an array with large number of elements
    • Knowledge
      • Describe different big o notation and their meanings
    • Application
      • Talk about the average, best and worst case for the sorting algorithms
    • Evaluation

  • Describe how would bubble sort sort
    • Compare each pair of adjacent elements
    • If they are not in correct order then swap elements
    • If they are in correct order then don’t swap
    • Return to the start when you read the end of the array
    • Set a flag to be true whenever a swap is made
    • repeat the loop if the flag is true
