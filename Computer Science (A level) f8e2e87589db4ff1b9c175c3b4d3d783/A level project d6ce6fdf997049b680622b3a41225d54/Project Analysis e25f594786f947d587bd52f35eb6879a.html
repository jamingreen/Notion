Project Analysis

Project Overview

This a Mario-like platform game. The player will control a character of their choice to pass through a series of obstacles and enemies to reach the final destination for the completion of the level.

It will have 3 level with different map and layout. The map will be predetermined and imported locally. There will be a time limit and faster finishing time will have a higher final score. There will be a local database storing all the previous entry and record of the game.

Why is the project needed?

The similar game (Super Mario Bros) is created long time ago. It is needed to create a similar game with higher image quality and more compatible to common modern machines.

I want to create this game is because I want to recreate one of the most well-known classic game in the history.

Computational Solution

It is to provide entertainment to the users. A computational solution is required as a similar game in real life is not possible or it is very costly. It is also dangerous to recreate it in real life.

Moreover, the original game is a computer game.

Research

Similar Game: Super Mario Bros

They use a moving background map while keeping the player character still in most of the time. It is a predetermined stage with same map layout, enemy movement and reward. There is no randomness in the game. The same series of input in the same time will result in the same outcome in the game.

I can use the moving background but not character mechanism. The map will be predetermined and loaded locally from a flat file

My solution will have a little randomness in the reward and tools drop rate in the game

This game have a very small initial size of 32kb, so the approximate size of the game result will be less than 100Mb since we use a higher resolution image and map.

Stakeholder

The stakeholder of the project is my teacher as it is necessary to grade my A level result based on this.
